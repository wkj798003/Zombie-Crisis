# 项目代号：Project Z² — 完整游戏设计文档

> **版本:** v1.0  
> **引擎:** Godot 4.7 (Mobile 渲染器, Jolt Physics)  
> **目标平台:** PC (Windows) 优先, 可横向扩展至主机/移动端  
> **单局时长:** ~15 分钟  
> **最后更新:** 2026-06-21

---

## 1. 玩法概览与核心循环

### 1.1 类型定位

> **2.5D Roguelite 射击游戏**  
> 视角采用**俯视/斜45度混合**的 2.5D 呈现——角色和怪物为平面 Sprite，场景使用多层视差 parallax 营造深度感。操作手感接近俯视角射击 (Top-down Shooter)，但视觉上有立体层次。

### 1.2 核心循环

```
移动探索 → 遭遇敌人 → 瞄准 & 射击 → 规避 & 走位
	 ↑                                        ↓
	 ↑                                   击杀敌人
	 ↑                                        ↓
	 结算 ←—— 15分钟倒计时结束 —— 成长选择 ← 获得 XP → 升级阈值到达
```

**一句话循环:** 击杀僵尸 → 获取经验 → 升级 → 选择"新武器"或"强化现有武器" → 切枪确认 → 继续战斗。

### 1.3 单局节奏 (15 分钟拆解)

| 时间段 | 阶段名称 | 玩家状态 | 设计目标 |
|--------|----------|----------|----------|
| 00:00–02:00 | **开局期** | 仅初始小手枪 | 让玩家熟悉移动/射击/规避；低强度僵尸 2-3 只一波 |
| 02:00–06:00 | **成长期** | 首次升级到来 | 玩家获得第一个选择（新武器或强化）；敌人密度略升 |
| 06:00–10:00 | **发力期** | 2-3 次升级后 | 组合武器与强化形成战斗力；混合敌人类型出现 |
| 10:00–13:00 | **高压期** | 高密度 + 高威胁敌人 | 玩家需要成熟的走位与武器组合；资源压力最大 |
| 13:00–15:00 | **终局冲刺** | 满配或濒死 | 大量敌人冲刺，经验/奖励翻倍，结算倒计时压力 |

> **关键设计约束:** 15 分钟必须是硬截止——时间到即进入结算，不论玩家状态。这塑造了"限时肉鸽"的独特节奏，迫使玩家在每一秒做有效决策。---

### 2.2 射击手感参数

**核心原则:** 单发小手枪的"按下即发、短冷却"风格。

```yaml
# 小手枪参考参数
pistol:
  fire_rate: 0.45         # 射击间隔 (秒)，即 0.45s 冷却
  bullets_per_shot: 1
  damage: 12
  crit_chance: 0.1        # 10% 暴击率
  crit_multiplier: 2.0
  spread: 0.03            # 基础散布 (弧度)
  recoil_vertical: 0.02
  recoil_horizontal: 0.01
  ammo_max: 12
  ammo_per_shot: 1
  reload_time: 1.5        # 秒
  bullet_speed: 800       # 像素/秒
```

**辅助瞄准 (右键):**  
- 开启后移动速度降低 40%  
- 散布缩小 50%  
- 摄像机轻微拉近  
- 适用于中远距离精准射击

**冲刺 / 翻滚 (Shift):**  
- 固定距离 120 像素  
- 0.3 秒无敌帧  
- 冷却 2.5 秒  
- 可打断射击后摇

## 3. 成长 / 升级系统

### 3.1 经验与升级阈值

采用**递增式经验曲线**，鼓励玩家持续击杀获取更高价值目标。

| 等级 | 所需 XP | 累计 XP | 备注 |
|------|---------|---------|------|
| 1→2 | 100 | 100 | 第一波小僵尸后即可升级 |
| 2→3 | 180 | 280 | |
| 3→4 | 300 | 580 | |
| 4→5 | 460 | 1040 | |
| 5→6 | 680 | 1720 | 中期节点 |
| 6→7 | 960 | 2680 | |
| 7→8 | 1300 | 3980 | |
| 8→9 | 1700 | 5680 | |
| 9→10 | 2200 | 7880 | 上限等级，满级后 XP 转化为分数加成 |

> **公式:** `xp_needed(level) = 10 x level² + 70 x level + 20`  
> 整数取整，视觉上清晰易读。

### 3.2 升级选择界面

升级弹出时游戏**暂停**（局部时间缩放至 0.05），玩家必须在 8 秒内做出选择，超时默认选择左项。

**两种分支类型:**

| 分支 | 内容 | 出现概率权重 |
|------|------|-------------|
| **A — 新武器** | 从当前尚未获得的武器池中随机抽 1 把 | 50% |
| **B — 强化当前武器** | 从当前装备武器的可用升级路径中随机抽 1 项 | 50% |

> **后续版本扩展:** 增加稀有度权重（绿/蓝/紫/金），高品质选项出现概率随等级提高。

### 3.3 拾取物 / 消耗品

战斗中通过击杀掉落或场景拾取获得：

| 拾取物 | 效果 | 掉落权重 | 堆叠上限 |
|--------|------|---------|----------|
| XP 小球 (小) | +10 XP | 高 | — |
| XP 宝石 (中) | +50 XP | 中 | — |
| XP 水晶 (大) | +200 XP | 低 | — |
| 生命补给 | +15 HP | 中 | — |
| 弹药箱 | 填满当前武器弹药 | 中 | — |
| 临时攻速 (拾取) | +30% 射速持续 8s | 低 | — |
| 临时护盾 | 抵挡下一次伤害 | 低 | — |

## 4. 武器系统

### 4.1 武器池

| 武器 | 解锁条件 | 基础伤害 | 射速(s) | 弹匣 | 特性 |
|------|---------|---------|---------|------|------|
| 初始手枪 | 默认 | 12 | 0.45 | 12 | 单发、高精度 |
| 冲锋枪 SMG | 任意升级选择 | 8 | 0.08 | 30 | 全自动、散布大 |
| 霰弹枪 | 等级≥2 可选 | 6×6 弹丸 | 0.70 | 6 | 近程爆发、装填慢 |
| 狙击步枪 | 等级≥3 可选 | 55 | 1.20 | 4 | 高穿透、开镜移速降低 |
| 双持手枪 | 等级≥2 可选 | 10×2 | 0.35 | 16 | 交替射击、换弹快 |
| 能量步枪 | 等级≥4 可选 | 18 | 0.25 | 20 | 穿透敌人、弹道显眼 |
| 榴弹发射器 | 等级≥5 可选 | 40 (范围) | 1.50 | 3 | 范围爆炸、友伤关闭 |

### 4.2 强化路径

每把武器有 2-3 条独立强化分支，每次升级选择该分支时推进一级。

**以初始手枪为例：**

```
强化路径 A — 制式升级
  Lv1: 伤害 +3
  Lv2: 弹匣容量 +4
  Lv3: 射速 0.45→0.35

强化路径 B — 战术改造
  Lv1: 暴击率 +5%
  Lv2: 暴击伤害 ×2.0→×2.5
  Lv3: 每次暴击回复 1 发弹药

强化路径 C — 穿透改装
  Lv1: 穿透 0→1 个敌人
  Lv2: 穿透后伤害 -30%→-15%
  Lv3: 穿透 +1 (共穿透 2 个)
```

### 4.3 武器切换规则

**核心约束：** 玩家**必须在进入射击前完成武器切换**。

```
战斗状态:
  1. 切换武器 (数字键/滚轮)
  2. 播放武器切换动画 (0.35s)
  3. 切换完成后才能射击
```

> **异常处理:** 如果玩家按下射击时正在切换动画中，不发射子弹且播放"咔"的空膛音效，提示切换未完成。

### 4.4 弹药管理

- 初始手枪弹药无限（但射速/弹匣约束依然存在）  
- 其他武器弹药有限，击杀僵尸有概率掉落弹药箱  
- 弹药箱拾取恢复当前武器 50% 弹匣量（至少一个弹匣）  
- 空枪时有 0.5s 的"空击"硬直惩罚

### 5.2 行为 AI 参数

```yaml
# 通用僵尸 AI 参数 (参考值)
ai:
  detection_radius: 400         # 索敌半径 (像素)
  hearing_radius: 300           # 声音触发半径
  lost_interest_time: 5.0       # 丢失目标后继续搜索时间 (秒)
  attack_range: 30              # 近战攻击判定距离
  attack_cooldown: 1.2          # 攻击间隔
  stagger_time: 0.35            # 受击硬直
  damage_to_player: 8
```

**精英/特殊行为标记:**
- Brute: `stagger_resistance = 0.7` (70% 概率抵抗硬直)
- Ghost: `teleport_cooldown = 2.5`, `invisibility = 0.4` (60% 透明)
- Commander: `buff_range = 250`, `attack_speed_buff = 0.25`, `move_speed_buff = 0.2`

### 5.3 生成规则 (Spawner)

```
每个房间的生成配置:
  wave_interval: 8-15 秒 (递减)
  enemies_per_wave: 2 + floor(game_time_minutes / 2)
  max_concurrent: 8 + floor(game_time_minutes / 1.5)
  
  敌人类别权重根据 game_time_minutes 动态调整:
	00-02: Walker 100%
	02-04: Walker 70%, Runner 30%
	04-06: Walker 40%, Runner 30%, Brute 15%, Thrower 15%
	06-08: +Exploder 15%, 基础敌人比例下调
	08-10: +Ghost, Commander 各 10%
	10+: 所有类型可用, 精英生成概率翻倍
```

## 6. 关卡与关卡生成

### 6.1 2.5D 场景结构

```
层次结构 (从近到远):
  [前景层]   — 装饰物、粒子遮挡 → Z-index 3
  [角色层]   — 玩家 + 敌人 + 可交互物 → Z-index 2 (主游戏层)
  [地面层]   — 地板 + 碰撞 → Z-index 1
  [背景层1]  — 近背景 (建筑废墟) → 视差 0.3x
  [背景层2]  — 远背景 (城市剪影) → 视差 0.1x
  [天空/雾层] — 色调雾 + 远景 → 视差 0x
```

**Godot 实现方式:**  
- 使用 `Parallax2D` 节点（Godot 4.x 内置） + `ParallaxLayer`  
- Y-sort 在角色层启用，确保角色和敌人正确前后遮挡  
- 碰撞使用 `CollisionPolygon2D` 或 `TileMap` 的物理层

### 6.2 房间生成器架构

```
关卡生成流程:
  1. 初始化种子 (每日种子 或 随机)
  2. 生成房间拓扑图 (3×3 网格 或 Procedural 图)
  3. 标记起始房间 (左上) 和终点房间 (右下)
  4. 填充每个房间的:
	 - TileMap 地板/墙壁
	 - 障碍物 / 掩体摆放
	 - 敌人出生点
	 - 可拾取物生成点
  5. 连接走廊 (简单 A* 或 L 型走廊)
  6. 放置特殊房间 (安全室 / 奖励室 / 精英房)
```

**房间类型分布:**

| 房间类型 | 权重 | 说明 |
|---------|------|------|
| 战斗房 | 60% | 标准敌人波次 |
| 精英房 | 15% | 1-2 精英敌人 + 高奖励 |
| 奖励房 | 10% | 无敌人，包含额外补给 |
| 安全房 | 10% | 起始/中途休息区 |
| 陷阱房 | 5% | 环境陷阱 + 高密度敌人 |

**每局房间数量:** 6-9 个房间（含起始和结束区域），保证 15 分钟内有足够推进空间。

### 6.3 掩体与交互

- 掩体: 可破坏（血量 100-300）和不可破坏两种  
- 通过掩体间的空格形成射击窗口  
- 陷阱: 地面尖刺、燃油桶爆炸、电击地板（对敌我都有效）

## 7. 难度曲线

### 7.1 随时间线性增长的参数

```
t = game_time_minutes (0.0 - 15.0)

敌人血量倍率:      1.0 + t * 0.06    (15分钟时 ≈ 1.9x)
敌人伤害倍率:      1.0 + t * 0.04    (15分钟时 ≈ 1.6x)
敌人移速倍率:      1.0 + t * 0.015   (15分钟时 ≈ 1.225x)
生成间隔:          lerp(15, 6, t/15) (从 15s 降到 6s)
每波数量:          2 + floor(t / 2)  (从 2 到 9)
```

### 7.2 玩家成长对标

```
玩家期望成长曲线 (围绕 "体验良好" 校准):
  时间 02:00:  等级 1-2,  1 把武器
  时间 05:00:  等级 3-4,  1-2 把武器 + 若干强化
  时间 08:00:  等级 5-6,  2 把武器 + 若干强化
  时间 11:00:  等级 7-8,  2-3 把武器 + 各 2 级强化
  时间 14:00:  等级 9-10, 2-3 把武器 + 大量强化
```

> **平衡点:** 确保在 10 分钟节点时，玩家的综合战斗力刚好能应对基础难度 1.6 倍左右的敌人，不做"数值碾压"也不做"必死"。

## 9. 艺术风格与视觉要素

### 9.1 2.5D 实现方案 (Godot 专项)

**核心思路:** 使用纯 2D 节点系统模拟 3D 深度感，避免实际 3D 场景的开销。

```
Node 结构:
  MainScene (Node2D)
	+-- Background (Parallax2D)
	|   +-- SkyLayer      — 远景天空/雾图
	|   +-- CityLayer     — 城市废墟剪影 (parallax: 0.1)
	|   +-- RuinsLayer    — 近处建筑废墟 (parallax: 0.3)
	+-- GameWorld (Node2D)
	|   +-- Floor (TileMap)         — 地板瓦片
	|   +-- Walls (TileMap)         — 墙壁/碰撞
	|   +-- Obstacles (Node2D)      — 掩体/障碍物
	|   +-- YSort (YSort2D)         — 角色层级排序
	|   |   +-- Player (CharacterBody2D)
	|   |   +-- Enemies (Node2D)
	|   +-- Items (Node2D)          — 可拾取物
	|   +-- FX (Node2D)             — 粒子/特效
	+-- Foreground (Parallax2D)
		+-- OverlayLayer — 近景遮挡物 (parallax: 1.5)
```

**Y-Sort 技巧:**  
- 所有角色和敌人使用 `YSort2D` 自动根据 y 坐标排序  
- 角色脚下放置 Shadow Sprite 作为"锚点"确定 Y 序  
- 角色 Sprite 的 `centered` 设为底部中心

### 9.2 美术氛围关键词

| 维度 | 方向 |
|------|------|
| 色调 | 暗紫/暗蓝为主，橙红/霓虹作为光源色 |
| 光照 | 动态 2D 光照 (`PointLight2D` + `DirectionalLight2D` + Shadow Filter) |
| 纹理风格 | 像素/手绘混合，适度的粗糙感；角色带 2-3 帧待机动画 |
| 血腥程度 | 适中，粒子血迹+地面贴花（不追求极端写实） |
| 霓虹元素 | 墙面发光管线、霓虹招牌、发光液体（增强视觉引导） |

### 9.3 特效列表

| 特效 | 实现方式 | 参考参数 |
|------|---------|---------|
| 枪口火焰 | `GPUParticles2D` + 点光源闪烁 | 生命周期 0.08s, 粒子数 5-8 |
| 子弹弹道 | `Line2D` + 渐隐 | 宽 2px, 颜色随武器变化 |
| 击中火花 | `GPUParticles2D` | 生命周期 0.3s, 方向随机 |
| 僵尸死亡 | 上抛 Sprite 碎片 + 粒子血迹 | 碎片 3-5 片, 渐隐 0.5s |
| XP 拾取 | XP 小球飞向 UI XP 条 | 缓动 `ease_out` 0.3s |
| 升级光效 | 全屏闪白 0.1s + 粒子爆散 | 时间缩放缓冲 |
| 屏幕震动 | `Camera2D` 偏移动画 | 幅度 5-8px, 衰减 0.2s |

## 11. 性能与实现要点 (Godot 专项)

### 11.1 2.5D 性能策略

| 优化项 | 做法 | 预期收益 |
|--------|------|---------|
| **Sprite Atlas** | 合并角色帧到图集，减少 Draw Call | 高 |
| **TileMap 分层** | 地面/墙壁分 TileMap 层，不需碰撞的设为 `No Collision` | 中 |
| **粒子 LOD** | 当粒子数 > N 时降低发射率 | 高 |
| **视锥裁剪** | 超出屏幕 200px 的敌人暂停 AI | 高 |
| **Light2D Cull Mask** | 仅影响特定层，减少光照计算 | 中 |
| **texture_cache_size** | 适当调整 GPU 纹理缓存 | 低(对移动端重要) |
| **对象池** | 子弹、敌人、拾取物使用预分配池 | 高 |

### 11.2 对象池实现建议

```gdscript
# 轻量对象池接口
class_name ObjectPool
extends Node

var pool: Array[Node] = []

func get_object() -> Node:
	var obj = pool.pop_back()
	if not obj:
		obj = create_instance()
	obj.visible = true
	obj.process_mode = PROCESS_MODE_INHERIT
	return obj

func return_object(obj: Node) -> void:
	obj.visible = false
	obj.process_mode = PROCESS_MODE_DISABLED
	obj.position = Vector2.ZERO
	pool.append(obj)
```

### 11.3 碰撞与射线检测

```gdscript
# 使用 PhysicsDirectSpaceState2D 进行高效射线检测
func shoot(from: Vector2, direction: Vector2, range: float) -> Dictionary:
	var space_state = get_world_2d().direct_space_state
	var query = PhysicsRayQueryParameters2D.new()
	query.from = from
	query.to = from + direction * range
	query.collision_mask = COLLISION_MASK_ENEMY | COLLISION_MASK_WALL
	query.exclude = [self]
	return space_state.intersect_ray(query)
```

**碰撞层级建议:**

| 敌人 | 2 | 所有敌人 |
| 子弹 | 4 | 玩家/敌人的子弹 |
| 墙壁 | 8 | 不可穿过碰撞 |
| 掩体(可破坏) | 16 | Destructible |
| 拾取物 | 32 | Items |
